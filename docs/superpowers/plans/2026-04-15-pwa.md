# LifeStep PWA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convertir LifeStep en una PWA instalable en iPhone con soporte offline, desplegada en Vercel.

**Architecture:** Se añade un Service Worker manual en `public/sw.js` con estrategia Network-First que cachea el shell de la app. Se registra en `app/layout.tsx` con un script inline. El despliegue se hace conectando el repo de GitHub a Vercel.

**Tech Stack:** Next.js 16 App Router, Vercel (hosting gratuito), Service Worker API nativa (sin paquetes extra)

### Task 1: Crear el Service Worker

**Files:**
- Create: `public/sw.js`

- [ ] **Paso 1: Crear `public/sw.js`** con la siguiente implementación completa:

```js
const CACHE_NAME = 'lifestep-v1';

const CACHED_URLS = [
  '/',
  '/dashboard',
  '/quests',
  '/shop',
  '/profile',
  '/manifest.json',
  '/icons/icon-192.png',
  '/icons/icon-512.png',
];

// Al instalarse: guarda los archivos principales en caché
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME).then((cache) => cache.addAll(CACHED_URLS))
  );
  self.skipWaiting();
});

// Al activarse: elimina cachés de versiones antiguas
self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then((keys) =>
      Promise.all(
        keys
          .filter((key) => key !== CACHE_NAME)
          .map((key) => caches.delete(key))
      )
    )
  );
  self.clients.claim();
});

// Al recibir peticiones: intenta la red primero, si falla usa caché
self.addEventListener('fetch', (event) => {
  if (event.request.method !== 'GET') return;

  event.respondWith(
    fetch(event.request)
      .then((response) => {
        if (response.ok) {
          const clone = response.clone();
          caches.open(CACHE_NAME).then((cache) => cache.put(event.request, clone));
        }
        return response;
      })
      .catch(() => caches.match(event.request))
  );
});
```

### Task 2: Registrar el Service Worker en el layout

**Files:**
- Modify: `app/layout.tsx`

- [ ] **Paso 1: Leer el archivo actual**

Abrir `app/layout.tsx` y localizar el tag `<body>`.

- [ ] **Paso 2: Añadir el script de registro**

Dentro del `<body>`, justo antes del cierre `</body>`, añadir este bloque:

```tsx
<script
  dangerouslySetInnerHTML={{
    __html: `
      if ('serviceWorker' in navigator) {
        window.addEventListener('load', function() {
          navigator.serviceWorker.register('/sw.js');
        });
      }
    `,
  }}
/>
```

El `<body>` debe quedar así:

```tsx
<body className="bg-white font-sans antialiased">
  <main className="min-h-screen pb-16">
    {children}
  </main>
  <BottomNav />
  <script
    dangerouslySetInnerHTML={{
      __html: `
        if ('serviceWorker' in navigator) {
          window.addEventListener('load', function() {
            navigator.serviceWorker.register('/sw.js');
          });
        }
      `,
    }}
  />
</body>
```

### Task 3: Desplegar en Vercel

> Este paso es manual — no hay código que escribir.

- [ ] **Paso 1: Subir los commits a GitHub**

```bash
git push origin master
```

- [ ] **Paso 2: Crear cuenta en Vercel**

1. Ir a [vercel.com](https://vercel.com)
2. Hacer clic en **"Sign Up"**
3. Elegir **"Continue with GitHub"**
4. Autorizar Vercel en GitHub

- [ ] **Paso 3: Importar el proyecto**

1. En el dashboard de Vercel, hacer clic en **"Add New Project"**
2. Buscar el repositorio `LifeStep` en la lista
3. Hacer clic en **"Import"**
4. Dejar toda la configuración por defecto (Vercel detecta Next.js automáticamente)
5. Hacer clic en **"Deploy"**

- [ ] **Paso 4: Esperar el deploy**

El primer deploy tarda ~2 minutos. Cuando aparezca el confeti y la preview, la URL estará lista.

- [ ] **Paso 5: Anotar la URL**

La URL tendrá el formato `lifestep-[algo].vercel.app`. Esta es la URL que usarás en el iPhone.

### Task 4: Verificar PWA en navegador de escritorio

> Verificación antes de pasar al iPhone. Requiere Chrome en el ordenador.

- [ ] **Paso 1: Abrir la URL de Vercel en Chrome**

Navegar a `https://lifestep-[algo].vercel.app`

- [ ] **Paso 2: Abrir DevTools y verificar el Service Worker**

1. Presionar `F12` para abrir DevTools
2. Ir a la pestaña **Application** (puede estar oculta en `»`)
3. En el menú izquierdo, hacer clic en **Service Workers**
4. Verificar que aparece `sw.js` con estado **"activated and running"**

- [ ] **Paso 3: Verificar el Manifest**

1. En el menú izquierdo de Application, hacer clic en **Manifest**
2. Verificar que aparecen: nombre "LifeStep", iconos, `display: standalone`
3. No debe haber errores en rojo

- [ ] **Paso 4: Ejecutar Lighthouse**

1. Ir a la pestaña **Lighthouse** en DevTools
2. Seleccionar solo **"Progressive Web App"** en las categorías
3. Hacer clic en **"Analyze page load"**
4. El resultado debe mostrar checks verdes en "Installable" y "PWA Optimized"

Si hay errores en Lighthouse, verificar qué falla antes de continuar.

### Task 5: Instalar en iPhone

> Paso manual en el iPhone.

- [ ] **Paso 1: Abrir Safari** en el iPhone (no Chrome, no Firefox — solo Safari)

- [ ] **Paso 2: Navegar a la URL de Vercel**

Escribir la URL completa: `https://lifestep-[algo].vercel.app`

- [ ] **Paso 3: Añadir a pantalla de inicio**

1. Tocar el botón **Compartir** — el ícono de cuadrado con una flecha hacia arriba, en la barra inferior de Safari
2. Desplazarse en el menú hasta encontrar **"Añadir a pantalla de inicio"**
3. Cambiar el nombre si se desea (por defecto ya será "LifeStep")
4. Tocar **"Añadir"** en la esquina superior derecha

- [ ] **Paso 4: Verificar instalación**

1. Ir a la pantalla de inicio del iPhone — debe aparecer el ícono de LifeStep
2. Tocarlo — debe abrirse en pantalla completa sin la barra de Safari

- [ ] **Paso 5: Verificar modo offline**

1. Activar el modo avión en el iPhone
2. Abrir LifeStep desde el ícono de la pantalla de inicio
3. La app debe cargar completamente aunque no haya internet
4. Desactivar modo avión al terminar la prueba
